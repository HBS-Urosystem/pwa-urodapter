# What's Changed from the Scraped Version

This document tracks all changes between the original scraped site (`scraped/app.urodapter.com/`) and the new SvelteKit PWA. It must be kept up to date whenever modifications are made.

---

## Platform & Framework

| Aspect    | Original (scraped)                             | New (SvelteKit PWA)                                                                                                                                            |
| --------- | ---------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Framework | Laravel + jQuery + Bootstrap 4                 | SvelteKit + Svelte 5 (runes)                                                                                                                                   |
| CSS       | Bootstrap 4 + custom `app.css` (~11K lines)    | Tailwind CSS v4 + DaisyUI v5                                                                                                                                   |
| JS        | jQuery + Webpack bundle (`app.js`, ~24K lines) | SvelteKit with Vite                                                                                                                                            |
| Adapter   | Server-rendered (Laravel)                      | Netlify (`@sveltejs/adapter-netlify`); [`netlify.toml`](netlify.toml) sets `publish` to `build` and cache headers for `/_app/immutable/*` and the web manifest |
| PWA       | None (no service worker, no manifest)          | Full PWA with service worker, manifest, offline support                                                                                                        |

- On the home page, table-of-contents icon handling is now data-driven and normalized to consistent `24x24` inline SVGs, and the `Install App` action appears beside `Share` in the footer action row instead of as a card.
- The site-wide footer groups **Contact** and **Install App** as two adjacent icon-only links (envelope + download tray, Heroicons-style strokes) with `aria-label`s; Privacy Policy and Cookie Policy stay text links. The bottom row shows **v2.0** immediately after the Urodapter wordmark.
- The home TOC strip uses `bg-accent/20` from theme tokens (no hardcoded `dark:` background). Card icons use `text-accent`. Footer uses `bg-primary` / `dark:bg-base-300` instead of a hardcoded teal OKLCH fill.
- **Footer vs. main**: **`Footer.svelte`** is the **last block inside `<main>`** (wrapped in **`w-full shrink-0`** only—no **`mt-auto`**, which would inject a large flex gap). Content **`section`**s no longer use **`min-h-full`** now that the footer is inside **`<main>`**; **`min-height: 100%`** against **`main`** plus a footer sibling stretched the section and left a tall empty band. **`ContentFlowNav`** stays **`sticky bottom-0`** with its sticky range bounded by **`<main>`** (which includes the footer).
- **Manual flow navigation**: below the main content on **`/how-the-urodapter-works`**, **`/what-to-do-prior-to-instillation`**, both doctor instruction routes, and **`/faq`**, **`ContentFlowNav.svelte`** shows **previous** and **next** links in TOC order (Educational Video ↔ prior instillation ↔ instructions ↔ FAQ ↔ Table of Contents). The bar is rendered **as a sibling after each page’s `<section>`** (not inside **`max-w-3xl`**), so it is naturally **`w-full`** under **`<main>`**—no viewport **negative-margin** breakout. It uses **`sticky bottom-0 z-10`**, opaque **`bg-base-100`**, **`border-y border-base-300`** (top and bottom rules), **`py-1`**, **`px-6`** on the link row to align with the footer’s **`p-6`**, and flow pages use **`section`** **`pt-8 pb-10`** so the breathing room before the bar sits inside the **`bg-primary/20`** content band (not on the nav margin). **`CONTENT_FLOW_NAV`** in [`src/lib/data/content-flow-nav.ts`](src/lib/data/content-flow-nav.ts) defines labels and default paths; links **to** “Instructions for doctors” from **`/faq`** (back) and **`/what-to-do-prior-to-instillation`** (forward) use **`readLastInstructionsPath()`** in [`src/lib/instructions-tab-storage.ts`](src/lib/instructions-tab-storage.ts) so the same **Female / Male** route is restored as when the user last viewed instructions (`localStorage` key **`urodapter-instructions-last-gender-v1`**, written on mount of each doctor instructions page).
- **Equal-width two-tab rows**: **`/what-to-do-prior-to-instillation`** (Patient \| Doctor) and doctor instruction pages (**Female \| Male** in [`InstructionsPatientPage.svelte`](src/lib/components/InstructionsPatientPage.svelte)) use a **`tabs-box`** with **`flex w-full`** and each **`tab`** **`flex-1 basis-0`** so the two tabs split the row **50/50**.
- **Page `<h1>` titles**: **`text-3xl`** without **`font-bold`** (regular weight on home, legal, FAQ, contact, install, educational video, prior instillation, and doctor instruction pages).
- **Prior instillation Patient / Doctor tab**: [`/what-to-do-prior-to-instillation`](src/routes/what-to-do-prior-to-instillation/+page.svelte) restores the last **Patient** vs **Doctor** tab from **`localStorage`** (**`urodapter-prior-instillation-audience-v1`**) on mount and saves the choice when the user switches tabs ([`src/lib/prior-instillation-audience-storage.ts`](src/lib/prior-instillation-audience-storage.ts)). **Patient** and **Doctor** cards share the same structure: audience line **`For patients`** / **`For healthcare professionals`** as **`<h2 class="text-2xl !mt-4 mb-2">`**, then letter rows with **`min-w-0 flex-1`** wrapping **`prose max-w-none flex-1 text-base-content max-sm:prose-sm`**. [`DoctorPriorToInstillationPanel.svelte`](src/lib/components/DoctorPriorToInstillationPanel.svelte) renders each letter’s **`paragraphs`** via **`formatBlockMarkdown`** (joined with blank lines) and passes **`h3Class: 'text-lg !mt-4 mb-2'`** so **`###`** lines in **`doctor-prior-instillation.md`** (e.g. the empty-bladder lead under **A**) emit **`<h3>`** at **`text-lg`** without the patient-tab semibold default. Patient card **`###`** subheads still use default **`formatBlockMarkdown`** **`h3`** (**`text-lg font-semibold …`**) from [`markdown-blocks.ts`](src/lib/markdown-blocks.ts).

## New Features (not in original)

- **PWA / Offline support**: service worker with precache (app shell, images, fonts) and runtime video caching; `/assets/video/*` uses **cache-first** (cached response if present, otherwise fetch and store). Runtime `cache.put` only runs for full responses (not **206** partial / `Content-Range`), since the Cache API rejects partial responses when the browser uses `Range` for playback.
- **Download all videos**: home page button precaches all 23 instruction videos (~390MB). **Benefits**: full offline viewing after download; when still online, replays use the cached file—**less mobile data**, **faster starts**, **fewer stalls** on weak Wi‑Fi (same files are not re-downloaded each time).
- **PWA install guide**: `/install` page with platform-specific instructions (iOS, Android, desktop)
- **Step memory**: doctor instruction carousel stores the active **step** in `localStorage` so reload returns to the same place (e.g. step 4/11)
- **Keyboard navigation**: arrow keys navigate instruction carousel steps
- **Start over**: on the last instruction step, the primary control is **Start over** (not a disabled **Next**); it returns to the first step and updates `localStorage`
- **Instruction step carousel**: DaisyUI `carousel` + `carousel-item w-full` for each step; scroll position stays in sync with stored `stepIndex`; step videos use `preload="none"` so slides are not all fetched at once. Carousel width is measured via `ResizeObserver` and cached, so key/scroll handlers avoid repeated `clientWidth` reads that can trigger Lighthouse forced reflow warnings.
- **Update notification**: toast when a new service worker version is deployed; update checks now also run on app resume (`pageshow` / `focus` / `visibilitychange`) and by service-worker controller URL change detection so iOS installed app resumes still surface updates even if the SW `postMessage` was missed while backgrounded
- **Share / QR**: Web Share API button for easy clinic distribution
- **SEO / social**: [`SeoHead.svelte`](src/lib/components/SeoHead.svelte) on each route — `<title>`, `<meta name="description">`, `<link rel="canonical">`, Open Graph (`og:title`, `og:description`, `og:type`, `og:url`, `og:image`, `og:site_name`), and Twitter Card (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`). Default share image: `/icon-512.png`. Absolute URLs use **`PUBLIC_SITE_URL`** when set (Netlify env); otherwise the current request `origin` is used.
- **Sitemap / robots**: `GET /sitemap.xml` emits all public paths ([`src/lib/seo.ts`](src/lib/seo.ts) `SITEMAP_PATHS`); `GET /robots.txt` allows all crawlers and points to the sitemap (replaces the previous static `robots.txt` in `static/`).
- **Instruction videos**: silent clips use `muted`; native volume/mute controls are hidden in WebKit/Blink
- **Instruction video posters**: matching JPEG poster frames (`/assets/video/<id>.jpg`) are now wired through compiled content and used as `<video poster>` on educational and step videos
- **How to avoid leakage modal**: the female instructions `plus-9` dialog now renders a structured A/B/C layout with letter badges on the left and explanatory text on the right, matching the visual pattern used on `/what-to-do-prior-to-instillation`
- **Accessibility**: `aria-label` on videos, `playsinline`, proper heading hierarchy, keyboard-navigable instruction carousel

## Removed

- **Google Analytics**: `gtag.js` with tracking ID `G-9EBFHB2BPH` -- removed entirely. No analytics cookies, no third-party data transfers to Google.
- **Cookie consent banner**: not needed (only strictly necessary cookies remain)
- **jQuery**: entire jQuery dependency removed
- **Bootstrap 4**: replaced by Tailwind CSS + DaisyUI
- **CSRF tokens**: Laravel artifact, not applicable
- **Server-side search**: `POST /search` form removed (was server-dependent)
- **reCAPTCHA**: contact form now uses Netlify's built-in honeypot spam filtering
- **AJAX modal content**: `func_plus_data` modals loaded via `/showInstructionPlusDataModal/:id` — replaced by static copy under **`content/instructions/*/modals.md`**, compiled into the app (no runtime AJAX).
- **`ajaxSetFirstOpen`**: Laravel session tracking on first "Enter" click -- removed
